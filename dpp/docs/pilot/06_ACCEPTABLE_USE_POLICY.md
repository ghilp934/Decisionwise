# Acceptable Use Policy (AUP)

**Decisionproof API - 허용 사용 정책**

---

## 금지 행위

### 1. 공격 및 남용

**금지:**
- [ ] DDoS 공격, 서비스 거부 공격
- [ ] 무차별 대입 공격 (brute-force)
- [ ] SQL Injection, XSS 등 취약점 공격
- [ ] 보안 취약점 스캔 (사전 허가 없이)

**예외:** 보안 취약점 테스트는 사전 서면 동의 필요.

### 2. Rate Limit 우회

**금지:**
- [ ] Rate Limit 우회 시도 (다중 계정, IP 변조)
- [ ] 자동화 봇으로 과도한 요청
- [ ] 제한 초과 후 즉시 재시도 (Retry-After 무시)

**준수:**
- 60 RPM 제한 준수
- `Retry-After` 헤더 확인 후 재시도

### 3. API Key 공유 및 재판매

**금지:**
- [ ] API Key를 제3자와 공유
- [ ] API 접근 권한 재판매
- [ ] 다중 조직에서 1개 Key 공동 사용

**원칙:** 1 고객 = 1 API Key

### 4. 크롤링 및 대량 수집

**금지:**
- [ ] API를 이용한 데이터 크롤링 (허가 없이)
- [ ] 타사 서비스 미러링 목적 사용
- [ ] 경쟁 분석 목적 대량 수집

**예외:** 자사 서비스 개선 목적은 허용 (범위 내)

### 5. 불법 콘텐츠

**금지:**
- [ ] 불법 콘텐츠 생성/유통
- [ ] 저작권 침해 콘텐츠
- [ ] 명예훼손, 허위 정보 유포

---

## API 보안 위험 대응

### 인증/인가 오용

**금지:**
- [ ] 타인의 토큰 무단 사용
- [ ] 토큰 탈취 시도
- [ ] 권한 우회 (privilege escalation)

### 과다 요청

**금지:**
- [ ] 의도적인 서비스 과부하
- [ ] 무한 루프로 인한 요청 폭주
- [ ] Rate Limit 초과 후 지속 재시도

### 데이터 탈취

**금지:**
- [ ] 타 고객 데이터 접근 시도
- [ ] SQL Injection 등으로 DB 접근
- [ ] API 응답 데이터 무단 재배포

---

## 위반 시 조치

### 경고 (1차)

**조건:** 경미한 위반 (실수로 인한 Rate Limit 초과 등)

**조치:**
- 이메일 경고 발송
- 7일 내 시정 요구

### 일시 중단 (2차)

**조건:** 반복 위반 또는 중대 위반

**조치:**
- API Key 일시 정지 (24~72시간)
- 시정 계획서 제출 요구

### 영구 차단 (3차)

**조건:** 악의적 공격, 불법 콘텐츠, 반복 위반

**조치:**
- API Key 영구 무효화
- 계약 해지
- 법적 조치 (필요 시)

## 정상 사용 예시

**허용:**
- ✅ 자사 서비스에서 Decision API 호출
- ✅ Rate Limit 준수하며 반복 호출
- ✅ 오류 발생 시 `Retry-After` 준수 후 재시도
- ✅ 테스트 목적 소량 호출
