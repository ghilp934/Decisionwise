## 파일럿 종료 절차

### 정상 종료 (파일럿 기간 만료)

**절차:**
1. **종료 7일 전**: 이메일 알림 발송
2. **종료 1일 전**: 최종 확인 및 데이터 백업 안내
3. **종료일**: API Key 자동 만료
4. **종료 후 30일**: 데이터 보관 (백업 가능)
5. **종료 후 31일**: 데이터 영구 삭제

### 중도 해지

**고객 요청:**
- 방법: pilot-support@decisionproof.ai로 서면 요청
- 처리: 3영업일 내 확인 및 종료 일정 협의
- 환불: 04_BILLING_AND_REFUND.md 참조

**당사 귀책:**
- 중대 결함으로 서비스 불가 시 즉시 종료 가능
- 환불 및 보상 협의

## 데이터 백업 및 인수

### 백업 가능 데이터

**1. Runs 데이터**
```bash
# 파일럿 기간 중 생성한 모든 runs 내보내기
curl -H "Authorization: Bearer $TOKEN" \
  https://staging-api.decisionproof.ai/v1/runs?limit=1000 \
  > my_runs_backup.json
```

**2. 사용 로그**
- 요청: pilot-support로 로그 추출 요청
- 형식: CSV 또는 JSON
- 제공: 3영업일 내

### 백업 권장 시기

- **종료 7일 전**: 데이터 백업 시작
- **종료 1일 전**: 최종 백업 완료

## API Key 회수

### 자동 만료

**종료일 00:00 (KST):**
- API Key 자동 비활성화
- 이후 모든 요청: `401 Unauthorized`

### 수동 회수 (긴급)

보안 사고 등으로 긴급 회수 필요 시:
- 요청: pilot-support@decisionproof.ai
- 처리: 즉시 (영업시간 내)

## 계정 정리

### 종료 시 삭제 대상

**즉시 삭제:**
- API Key (비활성화)
- 활성 세션 (있는 경우)

**30일 보관 후 삭제:**
- Runs 데이터
- 로그 데이터 (90일 보관 정책 우선)

**영구 보관 (세법 준수):**
- 청구/결제 내역 (5년)

## 데이터 보관 및 파기

### 보관 기간

| 데이터 유형 | 보관 기간 | 근거 |
|------------|----------|------|
| Runs 데이터 | 종료 후 30일 | 백업 유예 |
| API 로그 | 90일 (일반 정책) | 운영 정책 |
| 지원 티켓 | 종료 후 30일 | 참조 목적 |
| 청구 내역 | 5년 | 세법 |

### 파기 방법

**자동 파기:**
- 스케줄러로 보관 기간 경과 시 자동 삭제
- 복구 불가능 (물리적 삭제)

**수동 파기 요청:**
- 고객 요청 시 즉시 파기 가능 (단, 법적 보관 의무 제외)
- 파기 증명서 발급 가능 (요청 시)

## Production 전환

### 파일럿 성공 시

**절차:**
1. 파일럿 평가 회의 (종료 전 2주)
2. Production 승인 신청
3. 별도 계약 체결
4. Production API Key 발급
5. 데이터 마이그레이션 (필요 시)

**주의:**
- Staging과 Production은 **완전 분리**
- Staging 데이터는 Production으로 자동 이전 안 됨
- 필요 시 수동 마이그레이션 (별도 협의)
